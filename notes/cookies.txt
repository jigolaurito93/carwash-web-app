‚úÖWhat are cookies?
    Supabase creates 3 main cookies when you log in:
        sb-[project]-auth-token     ‚Üí access_token (short-lived, ~1hr)
        sb-[project]-refresh-token  ‚Üí refresh_token (long-lived, days/weeks)  
        sb-[project]-provider-token ‚Üí OAuth tokens (if using Google/etc.)

‚úÖ await supabase.auth.refreshSession()
    What it does:
        Takes your current refresh token and exchanges it with Supabase Auth server to get new access token + refresh token pair.

    When it's used:
        Access token expired (usually 1 hour)
        You want to keep user logged in without forcing re-login
        Automatic in most cases (Supabase clients do it behind the scenes)

‚úÖ middleware.ts should now be called proxy.ts because of Next.js 16 (Canary)
‚úÖ How does middleware work?

    Browser ‚Üí makes HTTP request (click link, type URL, etc.)
        ‚Üì
    Next.js server ‚Üí "Hey, before handling this request, run proxy.ts first"
        ‚Üì
    Next.js ‚Üí calls your `proxy(request)` ‚Üí passes incoming HTTP request as argument
        ‚Üì
    Your proxy code runs ‚Üí `request` contains URL, headers, cookies, method, etc.
        ‚Üì
    You return `NextResponse` ‚Üí Next.js continues handling the request

‚úÖ Flow Diagram:

    1. User visits: http://localhost:3000/admin/dashboard
    ‚Üì
    2. Browser sends HTTP GET request to Next.js server
    ‚Üì  
    3. Next.js sees `proxy.ts` exists ‚Üí "Run proxy first!"
    ‚Üì
    4. Next.js calls: `proxy(request)` ‚Üê request = incoming HTTP request object
    ‚Üì
    5. Your code gets `request.nextUrl.pathname` = "/admin/dashboard"
    ‚Üì
    6. Your code returns `supabaseResponse` 
    ‚Üì
    7. Next.js uses your response ‚Üí serves `/admin/dashboard` page

‚úÖ What request contains:

    request.nextUrl.pathname      // "/admin/dashboard"
    request.nextUrl.searchParams   // URL query params
    request.cookies.getAll()       // Browser cookies (your Supabase auth cookies)
    request.headers.get('user-agent') // Browser info
    request.method                 // "GET", "POST", etc.

‚úÖ The Flow When Tokens Refresh:
    1. Browser sends request with OLD cookies
    2. getAll() ‚Üí Supabase sees expired access token  
    3. supabase.auth.getUser() ‚Üí "Token expired, need refresh"
    4. refreshSession() ‚Üí Supabase calls setAll(NEW_COOKIES)
    5. request.cookies.set() ‚Üí Request now has NEW cookies
    6. supabaseResponse = NextResponse.next(request) ‚Üí New response with updated request
    7. supabaseResponse.cookies.set() ‚Üí Response has NEW cookies  
    8. Browser receives response ‚Üí Now has NEW cookies for next request
